Permisos disponibles para su sitio web¶

A continuación se muestra una lista de permisos para las operaciones que usted puede realizar en su instalación de Plesk (se muestran en Cuenta > pestaña Permisos):

  • Administración de zonas DNS. Administre registros de recurso en las zonas DNS de los sitios web. DNS son las siglas de Domain Name System. Se trata de un servicio que permite a los navegadores web localizar sitios web a través de nombres de dominio.

    Si desea más información acerca de la configuración de los parámetros DNS para sus sitios web, consulte la sección (Avanzado) Configuración del DNS para un dominio.

  • Administración de configuración de hosting. Administre las opciones de hosting web, como por ejemplo las opciones personalizadas de servidor web o el soporte para lenguajes de scripting.

  • Administración de los parámetros de configuración de PHP comunes. Ajuste parámetros de configuración comunes de PHP, como por ejemplo safe_mode u open_basedir para sus sitios web.

  • Administración del acceso al servidor a través de SSH y Administración de acceso al servidor a través de Escritorio Remoto. Conéctese al servidor de forma segura para gestionar archivos y cargar contenido web a través de una conexión Secure Shell (Linux) o Remote Desktop (Windows).

  • Administración de FTP anónimo. Cree una carpeta en el servidor a la que puedan acceder los usuarios de Internet a través del protocolo FTP. Esta carpeta puede tener una dirección como ftp://downloads.example.com. Esta prestación se denomina FTP anónimo porque los usuarios no deben especificar ningún nombre de usuario o contraseña para examinar, descargar o cargar archivos.

    Si desea información acerca del procedimiento para la configuración de una carpeta FTP con acceso no restringido, consulte la sección Configuración de acceso FTP anónimo.

  • Administración del programador. Programe la ejecución de programas y scripts en el sistema operativo del servidor.

    Si desea más información acerca de la programación de tareas, consulte el capítulo Programación de tareas.

  • Administración del filtro antispam. Establezca parámetros de configuración personalizados para protegerse de correo comercial no deseado, también denominado spam.

    Si desea más información acerca de la configuración del filtro antispam, consulte la sección Protección antispam.

  • Administración del antivirus. Establezca una configuración personalizada para protegerse de viruses y otros softwares malintencionados que se propaga a través de los correos electrónicos.

    Si desea más información acerca de la configuración del filtro antivirus, consulte la sección Protección antivirus.

  • Copia y restauración de los datos de suscripciones usando el almacenamiento del servidor. Use el administrador de backups de Plesk para crear backups y restaurar los datos de una suscripción: sitios web, cuentas de correo, bases de datos, registros y estadísticas, páginas de error personalizadas, listas de correo, certificados SSL/TLS e información DNS. Guarde los archivos de backup en el servidor de Plesk.

    Si desea más información acerca de la creación de backups y la restauración de datos, consulte el capítulo (Avanzado) Creación de backups y restauración de sitios web.

  • Copia y restauración de los datos de suscripciones usando un almacenamiento FTP. Use el administrador de backups de Plesk para crear backups y restaurar los datos de una suscripción: sitios web, cuentas de correo, bases de datos, registros y estadísticas, páginas de error personalizadas, listas de correo, certificados SSL/TLS e información DNS. Guarde los archivos de backup en un directorio FTP del servidor especificado.

  • Copia y restauración de los datos de cuentas usando el almacenamiento del servidor. Utilice el administrador de backups de Plesk para copiar y restaurar sus sitios web y la información de su cuenta y guarde los backups en el almacenamiento del servidor Plesk. Tenga en cuenta que este tipo de backup no está disponible por omisión, para evitar confusiones con los backups de las suscripciones (que no contienen información de las cuentas).

  • Copia y restauración de los datos de cuentas usando un almacenamiento FTP. Utilice el administrador de backups de Plesk para copiar y restaurar los datos de suscripciones y cuentas y guarde los backups en servidores FTP externos. Tenga en cuenta que este tipo de backup no está disponible por omisión, para evitar confusiones con los backups de las suscripciones (que no contienen información de las cuentas).

  • Administración de estadísticas web. Establezca preferencias personalizadas para los informes estadísticos de visitas.

    Si desea más información acerca de la configuración y visualización de estadísticas de visitas a un sitio web, consulte la sección Visualización de estadísticas.

  • Administración de rotación de registros. Establezca preferencias personalizadas para el reciclaje (rotación) de los registros del servidor web. El servidor web registra la información acerca de las conexiones a sus sitios y de posibles errores al intentar recuperar archivos ausentes. Puede usar estos archivos de registro para finalidades de depuración de sitios web.

    Si desea más información acerca de los registros de acceso al servidor web, consulte la sección Archivos de registro.

  • Acceso al catálogo de aplicaciones. Vea e instale aplicaciones en sitios web. Si desea más información acerca de las aplicaciones, consulte la sección Aplicaciones web.

  • Acceso a WP Toolkit. Administre instancias de WordPress instaladas en sus sitios web mediante una interfaz de gestión muy fácil de usar. Si desea más información, consulte la sección WP Toolkit.

  • Administración de seguridad de WP Toolkit. Gestione las opciones de seguridad disponibles en WP Toolkit.

  • Configuración de opciones de script web potencialmente inseguras que sustituyan la directiva del proveedor. Le permite invalidar la directiva de seguridad de hosting, de ser aplicada por el proveedor.

  • Gestión de dominios. Cree y administre sitios web nuevos.

    Si desea información acerca de la creación y configuración de sitios web, consulte las secciones Creación de su primer sitio web y Adición de dominios.

  • Administración de subdominios. Cree y administre sitios web nuevos con direcciones como forum.example.com.

    Si desea más información acerca de la creación de subdominios, consulte la sección Adición de subdominios.

  • Administración de alias de dominio. Cree y administre nombres de dominio adicionales para un sitio.

    Si desea información acerca de la creación y configuración de alias de dominio, consulte el capítulo Adición de alias de dominio.

  • Administración de cuentas FTP adicionales. Configure y administre cuentas FTP adicionales. Para permitir la colaboración en el contenido de un sitio web, puede configurar cuentas FTP para otros usuarios y especificar a qué directorios del sitio pueden acceder estos usuarios.

    Si desea más información acerca de la creación de cuentas FTP, consulte la sección Adición de cuentas FTP.

  • Administración de aplicaciones en Java. Instale y administre aplicaciones en Java distribuidas en archivos comprimidos WAR y obtenidos a través de otros desarrolladores de aplicaciones o vendedores.

  • Habilidad para cambiar la configuración de correo. Gestione la configuración del servidor de correo, seleccione el cliente de webmail o active o desactive el servicio de correo para un dominio.

  • Administración de listas de correo. Configure y administre listas de correo.

    Si desea más información acerca de la creación y el uso de listas de correo, consulte la sección Uso de listas de correo.

  • Administración de los parámetros de configuración del rendimiento de hosting. Ajuste la configuración de rendimiento de PHP para un sitio web. Además, en Plesk para Windows, permite to limitar el uso del ancho de banda y el número de conexiones al sitio web.

  • Administración de grupos de aplicaciones IIS. Establezca preferencias personalizadas para el grupo de aplicaciones IIS (sólo disponible en cuentas de cliente basadas en Windows).

    Si desea información acerca de la creación de grupos de aplicaciones IIS, consulte la sección Creación de un grupo de aplicaciones IIS (Windows).

  • Administración de permisos adicionales de escritura/modificación. Establezca permisos adicionales de escritura/modificación para sitios web que usen bases de datos basadas en archivos (sólo disponible en cuentas de cliente basadas en Windows).

  • Asignación de cuota de disco duro. Edite las cuotas relacionadas con el espacio en disco, siempre que su cuenta de cliente soporte esta opción.

  • Selección del servidor de base de datos. Seleccione un servidor de base de datos para la creación de bases de datos, si dispone de múltiples servidores de base de datos.

  • Acceso remoto para usuarios de bases de datos. Establezca opciones de control de acceso personalizadas para las cuentas de usuario de una base de datos. Para más información, consulte la sección Configuración de reglas de acceso personalizadas.

  • Acceso a operaciones avanzadas - Copia de sitio web. Copie el contenido de uno de sus sitios web a otro sitio web o a un almacenamiento FTP. Para más información, consulte la sección Uso de un sitio de ensayo.

  • Gestión de directorios protegidos con contraseña. Proteja uno o más directorios de su sitio web mediante el establecimiento de contraseñas. Cuando los visitantes deseen acceder a los directorios protegidos, se les solicitará esta contraseña. Si desea más información al respecto, consulte la sección Restricción de acceso al contenido.

  • Posibilidad de gestionar cuentas de usuario adicionales. Administre cuentas de usuario adicionales.Si desea más información, consulte la sección Administración de cuentas de usuarios adicionales.

  • Gestión de permisos de escritura/modificación adicionales (Windows). Conceda permisos de acceso completo sobre las carpetas httpdocs y httpsdocs para su usuario de sistema.

  • Administración de grupos de aplicaciones IIS (Windows). Active o desactive un grupo dedicado de aplicaciones para su sitio web y establezca la cantidad máxima de potencia de CPU que podrá utilizar dicho grupo. Para más información, consulte la sección Creación de un grupo de aplicaciones IIS .
